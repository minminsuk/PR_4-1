# 중고차 허위매물 판별 시스템 최종보고서

작성일: 2026-04-04

## 1. 프로젝트 개요
본 프로젝트는 차량 정보 기반의 AI 가격 예측 결과와 사용자가 입력한 매물 가격을 비교하여 정상, 주의, 위험 상태를 판별하는 웹 서비스다.

핵심 목표는 다음과 같다.
- 중고차 매물의 가격 적정성 판단 보조
- 허위매물 의심 케이스 조기 탐지
- 실서비스 형태의 프론트-백엔드-AI 추론 파이프라인 구현

## 2. 시스템 아키텍처
- 프론트엔드: React + TypeScript + Vite
- 백엔드: Node.js + Express + TypeScript
- 입력 검증: Zod 스키마
- AI 추론: Python(Pytorch) 모델을 Node 서버에서 자식 프로세스로 실행
- 통신 방식: REST API

동작 흐름
1. 사용자가 프론트 화면에서 차량 정보를 입력한다.
2. 프론트가 예측 API로 요청을 전송한다.
3. 백엔드가 Zod로 요청값을 검증한다.
4. 백엔드가 Python 추론 스크립트를 실행한다.
5. 예측 결과와 판정 결과를 프론트로 반환한다.

## 3. 핵심 기능
- 차량 가격 예측
- 입력 가격 대비 차액, 차이율 계산
- 정상, 주의, 위험 판정 제공
- 인증 사용자 조회
- 로그아웃 처리
- 입력값 오류 및 추론 오류 응답 처리

## 4. API 명세
### 4.1 GET /api/auth/me
- 설명: 현재 인증 사용자 조회
- 응답: 사용자 정보 또는 null

### 4.2 POST /api/auth/logout
- 설명: 세션 쿠키 만료 처리
- 응답: {"success": true}

### 4.3 POST /api/cars/predict
- 설명: 차량 입력값 기반 AI 추론
- 성공 응답 필드
  - predicted_price
  - input_price
  - price_difference
  - price_difference_percent
  - confidence_score
  - verdict
  - verdict_color
- 오류 응답
  - 400: 요청값 유효성 오류
  - 500: 추론 실행 오류

## 5. 모델 및 추론 구조
- 모델 파일: best_model_v15.pth
- 설정 파일: model_config_v15.json
- 추론 코드: backend/inference.py
- Node 브리지: backend/dnn_predictor.ts

특징
- Node 서버가 Python 스크립트를 spawn으로 실행
- 표준입출력(JSON) 기반 IPC로 데이터 전달
- 타임아웃 가드로 장시간 추론 실패를 방지

## 6. 실제 예측 반영 변수 정리
현재 서버 모델 설정 기준으로 가격 예측에 사용되는 변수는 다음과 같다.

### 6.1 범주형 변수
- fuel
- brand
- model
- price_segment

### 6.2 수치형 변수
- price
- car_age
- mileage
- spec_hp
- spec_cc
- spec_efficiency
- insu_count
- option_count
- opt_썬루프
- opt_네비게이션
- opt_스마트키
- opt_통풍시트
- opt_가죽시트
- opt_열선시트
- opt_후방카메라

중요 참고
- 현재 추론 코드에서는 price 입력값을 모델 입력으로 직접 사용하지 않고, 학습 평균값으로 대체한다.
- 현재 추론 코드에서는 price_segment도 입력 가격이 아니라 기준 가격 기반으로 고정한다.
- 즉, 입력 가격은 최종 비교(차액, 차이율) 계산에 주로 사용된다.

## 7. 연식 입력 정책
- 서비스 입력은 제조연도(예: 2025년식) 자체가 아니라 차량 나이(car_age) 기준으로 처리한다.
- 따라서 모델은 차량 나이를 이용해 가격 영향을 학습 및 추론한다.
- 제조연도 기반 UX가 필요하면 내부에서 차량 나이로 변환하는 레이어를 추가하면 된다.

## 10. 한계 및 개선 과제

현재 단계에서의 핵심 한계는 모델 구조 자체보다 학습용 데이터와 테스트용 데이터의 절대 규모 및 다양성이 충분하지 않다는 점이다. 현재는 보배드림 중고차 사이트에서 크롤링을하여 데이터를 수집했지만 중고차 사이트 특성상 제대로된 옵션및 차량정보를 적어놓지 않은 매물들 다수 존재 이로 인해 실제 시장의 다양한 매물 패턴을 충분히 일반화하기 어려워, 최고 수준의 예측 성능을 안정적으로 확보하는 데 제약이 있다.

개선 과제는 다음과 같다.
- 학습 데이터 확장: 차종, 연식, 주행거리 구간, 사고 이력, 옵션 조합이 고르게 분포되도록 데이터 규모를 단계적으로 확대
- 테스트 데이터 고도화: 실제 운영 환경을 반영한 독립 테스트셋을 추가 구축하여 일반화 성능을 정밀 검증
- 데이터 품질 관리: 이상치, 결측치, 라벨 편차를 정제하는 데이터 품질 기준을 수립하고 주기적으로 점검
- 검증 체계 강화: 단일 분할 검증을 넘어 교차검증 및 기간 분리 검증을 도입해 성능 신뢰도를 향상
- 재학습 파이프라인 정착: 신규 데이터 누적 시 재학습, 재평가, 배포를 반복하는 운영형 프로세스 구축

요약하면, 향후 성능 고도화의 핵심은 시스템 구조 변경보다 데이터 자산의 양적·질적 확장에 있다.
